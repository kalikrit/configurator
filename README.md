# Конфигуратор ПК — прототип

## Описание

Прототип конфигуратора персонального компьютера с динамической проверкой совместимости компонентов. Пользователь собирает ПК, перетаскивая компоненты из левой панели в правую. Система автоматически блокирует несовместимые компоненты и визуально показывает причину.

## Демонстрируемые возможности

- Выбор типа устройства (ПК, с заглушками для будущих устройств)
- Drag & Drop компонентов из каталога в сборку
- Динамическая фильтрация совместимости:
  - Процессор ↔ Материнская плата (по сокету)
  - Видеокарта ↔ Блок питания (по рекомендуемой мощности)
- Визуальное отображение доступных/недоступных компонентов:
  - Доступные: белый фон, зеленая рамка, активны для перетаскивания
  - Недоступные: серый фон, бейдж "⛔ Недоступно", заблокированы
- Удаление компонентов из сборки
- Зеленая рамка и сообщение при завершении сборки
- Итоговая стоимость

## Технологии

- Vue 3 (Composition API, `<script setup>`)
- Pinia (управление состоянием)
- Vite (сборка)
- Нативный Drag & Drop API

## Правила совместимости

1. **Процессор ↔ Материнская плата**
   - Сокет процессора должен совпадать с сокетом материнской платы
   - Пример: CPU с сокетом AM5 несовместим с материнской платой AM4

2. **Видеокарта ↔ Блок питания**
   - Мощность блока питания должна быть ≥ рекомендуемой мощности видеокарты
   - Пример: RTX 4090 требует БП ≥ 850W

## Как пользоваться

1. На стартовом экране выберите "Персональный компьютер (ПК)"
2. Перетащите процессор в правую панель
3. Перетащите материнскую плату (доступны только с совместимым сокетом)
4. Перетащите видеокарту
5. Перетащите блок питания (доступны только с достаточной мощностью)
6. Когда все 4 слота заполнены — появится зеленая рамка и сообщение об успехе
7. Любой компонент можно удалить кнопкой ✕

## Визуальная обратная связь

- **Доступные компоненты** в левой панели: белый фон, зеленая рамка, активный курсор
- **Недоступные компоненты**: серый фон, бейдж "⛔ Недоступно", заблокированы
- **При наведении на недоступный** — всплывающая подсказка
- **Собранная конфигурация** — зеленая рамка вокруг правой панели

## Расширяемость

Архитектура позволяет легко добавлять:

1. **Новые устройства** (сервер, ноутбук, рабочая станция)
   - Добавить конфигурацию в `config/devices.js`
   - Описать слоты и правила совместимости

2. **Новые слоты** для существующего устройства
   - Добавить слот в `slots` массиве
   - Добавить правило в `compatibilityRules`

3. **Новые компоненты**
   - Добавить в соответствующий массив в `data/components.js`

4. **Множественные компоненты** (несколько планок памяти, несколько накопителей)
   - Требует доработки store и UI

## Изменения в версии 1.8.0 (31.03.2026)

### Выделение слоя сервисов (Service Layer)
- **CompatibilityService** — сервис для проверки совместимости компонентов:
  - `checkSocketCompatibility()` — проверка соответствия сокетов CPU и материнской платы
  - `checkChipsetCompatibility()` — расширенная проверка совместимости чипсета
  - `checkMemoryTypeCompatibility()` — проверка типа памяти (для будущего расширения)
  - `checkAllRules()` — проверка всех правил совместимости
  - `getIncompatibilityReason()` — получение причины несовместимости
  
- **PowerService** — сервис для расчета мощности и подбора БП:
  - `calculateTotalPower()` — расчет общего энергопотребления системы
  - `isPsuSufficient()` — проверка достаточности мощности БП
  - `checkGpuRecommendedPsu()` — проверка соответствия рекомендациям видеокарты
  - `findSuitablePsu()` — автоматический подбор подходящего БП из списка
  - `getPowerStatus()` — получение статуса проверки мощности с детальным сообщением

### Улучшения store
- **Рефакторинг buildStore.js** — логика проверки совместимости делегирована CompatibilityService
- **Новые вычисляемые свойства**:
  - `powerInfo` — информация о потребляемой мощности системы
  - `powerStatus` — статус проверки блока питания (error/warning/success)

### Преимущества новой архитектуры
- **Разделение ответственности** — бизнес-логика отделена от состояния Vue
- **Тестируемость** — сервисы можно тестировать независимо от Vue
- **Переиспользуемость** — сервисы можно использовать в других проектах
- **Чистота кода** — store стал проще и понятнее
- **Подготовка к масштабированию** — легко добавлять новые сервисы

### Технические изменения
- Создание `src/services/CompatibilityService.js`
- Создание `src/services/PowerService.js`
- Обновление `src/stores/buildStore.js` — импортирование и использование сервисов
